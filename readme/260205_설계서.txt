========================================
프로젝트 설계서
========================================

프로젝트명
Mini Task Manager SPA

프로젝트 목적
비동기 로그인부터 태스크 CRUD까지 이어지는 SPA 구조를 구현한다.
전역 상태 관리와 라우터 param을 조합한 실무형 아키텍처를 학습한다.

프로젝트 범위
- 로그인 기능
- 태스크 목록 조회
- 태스크 생성, 수정, 삭제
- 태스크 상세 페이지
- 인증 기반 라우팅 제어

아키텍처 개요
- Frontend SPA 구조
- Mock API를 통한 비동기 통신
- 전역 상태 관리 중심 설계
- 페이지 라우팅과 상태 흐름 분리

데이터 흐름
Mock API
→ 전역 상태(store)
→ 페이지 컴포넌트
→ 자식 컴포넌트

인증 흐름
로그인 성공
→ 전역 상태 isLoggedIn 변경
→ 보호된 라우트 접근 허용

========================================
요구사항 정의서
========================================

1. 로그인
- 사용자는 로그인 버튼을 통해 로그인할 수 있어야 한다.
- 로그인은 비동기로 처리되어야 한다.
- 로그인하지 않은 사용자는 태스크 페이지에 접근할 수 없다.

2. 태스크 목록
- 로그인 후 태스크 목록을 조회할 수 있어야 한다.
- 태스크 목록은 Mock API를 통해 조회된다.

3. 태스크 생성
- 사용자는 새로운 태스크를 생성할 수 있어야 한다.
- 생성된 태스크는 즉시 목록에 반영되어야 한다.

4. 태스크 수정
- 사용자는 태스크 상태를 변경할 수 있어야 한다.
- 상태 변경은 상세 페이지에서 가능해야 한다.

5. 태스크 삭제
- 사용자는 태스크를 삭제할 수 있어야 한다.
- 삭제 후 목록에서 제거되어야 한다.

6. 태스크 상세
- 태스크 상세 페이지는 URL param(id)을 기반으로 접근한다.
- 새로고침 시에도 동일한 태스크를 조회할 수 있어야 한다.

7. 라우팅
- 페이지는 SPA 라우팅으로 동작해야 한다.
- 로그인 상태에 따라 접근 제어가 되어야 한다.
